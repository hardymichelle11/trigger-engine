# Session Notes — 2026-04-15

## Session 27: Credit-Vol Regime Pipeline + Validation Fix

### Problem
Regime card rendering HYG 0.00, KRE 0.00, VIX 0.00, LQD — despite valid live prices visible in chart data. Root cause was a cascade of 6 interrelated bugs in the data pipeline, engine, and UI.

### Root Causes Found
1. **VIX/TNX symbol mapping**: Polygon proxy plan lacks index data access. `fetchRegimeSnapshot` tried bare `VIX` against indices endpoint → 404. App.jsx used `I:VIX` via aggs endpoint but CreditVolScanner never did.
2. **`_invalidDataResult` undefined**: Engine called a function that didn't exist → crash when VIX data was missing → entire `refresh()` failed silently.
3. **LQD excluded from scoring**: Config had LQD weight (0.15) but `computeComposite`, feature extraction, explanation builder, and `indicators.lqd` all excluded it. `indicators.lqd` was hardcoded to `null`.
4. **`|| 0` coercion in composite**: `(scores.HYG || 0)` treats legitimate score of 0 as falsy.
5. **`day.c || prevClose` coercion**: When `day.c = 0` (after hours), `0 || prevClose = prevClose`. Then `dayClose !== prevClose` check skipped the push, leaving history at 1 bar → feature extraction defaulted to `{ current: 0 }`.
6. **History deduplication killed flat days**: When `dayClose === prevClose`, the dedup check dropped the second bar below the engine's 2-bar minimum.

### What Was Built

**Engine** (`creditVolRegimeV2.js`):
- `_dataUnavailableResult(diagnostics)` — returns mode `DATA_UNAVAILABLE`, null scores, `allowNewTrades: false`, per-symbol diagnostics
- `strictParseNumeric()` — rejects null/undefined/NaN without silent coercion
- `validateRegimeValue()` — price proxies reject 0 and negative
- `validateRegimeData()` — per-symbol diagnostics with freshness/staleness (6h threshold)
- LQD added to: feature extraction, `scoreDownRisk`, `computeComposite`, `buildExplanation`, `indicators`, `percentiles`
- `|| 0` → `?? 0` in `computeComposite`
- `dataAvailable` flag + `diagnostics` object in all engine output

**Data Fetching** (`CreditVolScanner.jsx`):
- `REGIME_SYMBOL_MAP` with `VIX → I:VIX`, `TNX → I:TNX`
- `fetchIndexSnapshot()` — routes VIX/TNX through worker `/index/` endpoint (Yahoo Finance)
- Polygon snapshot fallback chain: snapshot → prev aggs
- Source tracking metadata (`_source`, `_fieldUsed`, `_timestamp`) through pipeline
- Strict null checks replacing `||` coercion in `buildRegimeHistory`
- History always pushes dayClose when valid, even when equal to prevClose (2-bar minimum)
- Timestamps passed through to engine for freshness validation

**Worker** (`polygon-proxy.js`):
- `/index/:symbol` endpoint — fetches VIX/TNX from Yahoo Finance (`query2.finance.yahoo.com`), returns Polygon-compatible snapshot shape
- `YAHOO_SYMBOL_MAP`: VIX → `^VIX`, TNX → `^TNX`
- 60s cache headers, proper CORS
- Worker deployed: version `e95e9437`

**UI** (`CreditVolScanner.jsx`):
- `SymbolStatusBadge` component — LIVE (green), STALE (amber), MISSING (red), FALLBACK (purple)
- DATA UNAVAILABLE state: "MISSING INPUTS" mode badge, "DATA UNAVAILABLE" bias, null score bar, N/A status
- Indicator display uses strict `!= null && > 0` instead of truthy coercion
- Credit/Vol/Fear status shows "MISSING INPUTS" / "N/A" when data unavailable

**Tests** (`test-regime-v2.js`):
- Case 6: Null/undefined payloads, zero value rejection (non-positive), negative rejection, mixed valid/invalid inputs, no score emitted when required inputs invalid
- Case 7: Fallback_close detection (single-bar prevClose-only series)
- Case 8: LQD scoring verification (included in componentScores, indicators populated)
- Case 9: No confident bias with invalid data (NaN → DATA_UNAVAILABLE, not fake OK/CONTAINED/LOW)
- All existing tests updated to include LQD in input data
- Config test updated: 7 weights (was 6), sum still ~1.0
- **86 total assertions, 0 failures**

### Decisions Made
- VIX/TNX routed through Yahoo Finance (via worker proxy) instead of Polygon — Polygon plan lacks index data access
- VIX shows STALE badge after hours — honest labeling of data freshness rather than pretending it's live
- LQD is a required input (alongside HYG, KRE, VIX) — engine refuses to score without it
- `DATA_UNAVAILABLE` mode blocks all downstream scoring and trade signals (`allowNewTrades: false`)
- Regime panel never displays confident bias (OK/CONTAINED/LOW) when underlying inputs are missing
